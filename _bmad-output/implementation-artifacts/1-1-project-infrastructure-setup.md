# Story 1.1: Project Infrastructure Setup

Status: in-progress

## Story

As a developer,
I want the core project infrastructure (Convex, Clerk, shadcn/ui) installed and wired together,
so that all subsequent features have a working foundation to build on.

## Acceptance Criteria

1. Convex is installed and initialized with a `convex/` directory and dev deployment
2. Clerk is installed with environment variables configured
3. shadcn/ui is initialized with Tailwind CSS 4 integration
4. `ConvexProviderWithClerk` wraps the app in the root layout
5. Clerk middleware protects all routes except `/sign-in` and `/sign-up`
6. The Convex schema defines the `users` table with fields: `clerkId`, `name`, `email`, `role`, `createdAt`
7. A reusable `requireRole()` helper function exists for Convex mutation/query authorization
8. The app renders without errors on `localhost`

## Tasks / Subtasks

- [ ] Install and initialize Convex (AC: #1)
  - [ ] `npm install convex`
  - [ ] `npx convex dev --once` to initialize `convex/` directory and create dev deployment
  - [ ] Add Convex env vars to `.env.local` (`CONVEX_DEPLOYMENT`, `NEXT_PUBLIC_CONVEX_URL`)
  - [ ] Create `convex/schema.ts` with `users` table definition
- [ ] Install and configure Clerk (AC: #2)
  - [ ] `npm install @clerk/nextjs`
  - [ ] Create Clerk application in dashboard, configure email/password + social login
  - [ ] Add Clerk env vars to `.env.local` (`NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY`)
  - [ ] Add Clerk env vars for sign-in/sign-up redirect URLs
- [ ] Initialize shadcn/ui (AC: #3)
  - [ ] `npx shadcn@latest init` — select Tailwind CSS 4, New York style
  - [ ] Verify `components/ui/` directory created
  - [ ] Install initial components: `npx shadcn@latest add button input select toast table`
- [ ] Wire ConvexProviderWithClerk in root layout (AC: #4)
  - [ ] Create `app/providers.tsx` with `ConvexProviderWithClerk` wrapping
  - [ ] Update `app/layout.tsx` to use `<ClerkProvider>` and `<ConvexProvider>`
  - [ ] Verify auth token is passed from Clerk to Convex via `useAuth`
- [ ] Create Clerk middleware (AC: #5)
  - [ ] Create `middleware.ts` at project root
  - [ ] Configure `clerkMiddleware()` with `createRouteMatcher` for public routes (`/sign-in(.*)`, `/sign-up(.*)`)
  - [ ] Verify protected routes redirect to sign-in
- [ ] Define Convex users schema (AC: #6)
  - [ ] In `convex/schema.ts`, define `users` table: `clerkId` (string, indexed), `name` (string), `email` (string), `role` (optional union: "admin" | "manager" | "staff"), `createdAt` (number)
  - [ ] Add index on `clerkId` for fast lookup: `.index("by_clerkId", ["clerkId"])`
- [ ] Create requireRole() helper (AC: #7)
  - [ ] Create `convex/lib/auth.ts` with `requireRole(ctx, roles)` function
  - [ ] Helper reads user identity from `ctx.auth.getUserIdentity()`
  - [ ] Looks up user in `users` table by Clerk subject ID
  - [ ] Throws `ConvexError({ code: "UNAUTHORIZED" })` if role doesn't match
  - [ ] Returns the authenticated user document for downstream use
- [ ] Verify app renders (AC: #8)
  - [ ] Run `npm run dev` and `npx convex dev` concurrently
  - [ ] Verify app loads at `localhost:3000` without errors
  - [ ] Verify sign-in redirect works for unauthenticated users

## Dev Notes

- **Architecture pattern:** Incremental addition to existing Next.js 16 project (NOT a fresh starter template). Reference `get-convex/template-nextjs-clerk-shadcn` for wiring patterns only.
- **Convex dev workflow:** Run `next dev` + `npx convex dev` concurrently. Consider adding a `"dev"` script in package.json: `"concurrently \"next dev\" \"convex dev\""`
- **Convex schema validators:** Use `v.string()`, `v.optional(v.union(v.literal("admin"), v.literal("manager"), v.literal("staff")))` for role field
- **requireRole pattern:** Should accept single role string or array of roles. Return the user doc so callers don't need a second lookup.
- **ConvexProviderWithClerk:** Import from `convex/react-clerk`. Use `useAuth` from `@clerk/nextjs` as the auth prop.
- **Clerk middleware:** Use `clerkMiddleware()` from `@clerk/nextjs/server` with `createRouteMatcher` for public routes.
- **shadcn/ui with Tailwind 4:** Ensure `components.json` is configured for the project's CSS variable approach.

### Project Structure Notes

```
app/
  layout.tsx              # Root layout with ClerkProvider + ConvexProvider
  providers.tsx           # ConvexProviderWithClerk component
  (auth)/sign-in/page.tsx # Clerk sign-in (public)
  (auth)/sign-up/page.tsx # Clerk sign-up (public)
middleware.ts             # Clerk middleware at project root
convex/
  _generated/             # Auto-generated by Convex
  schema.ts               # Database schema (users table)
  lib/
    auth.ts               # requireRole() helper
components/
  ui/                     # shadcn/ui components
```
